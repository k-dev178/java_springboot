# 클래스
변수와 함수 보관하는 통

## 선언
```java
class Test {
   class Test{
    String name = "ksj";
    void hello(){
        System.out.println("안녕");
    }
  }
}
```

## 사용 - 불러오기
```java
public static void main(String[] args){
  var test = new Test(); // Test test = new Test();
}
```

이렇게 하면 클래스가 불러와짐.
클래스는 쓸려면 어딘가에 복사해야해서 test 라는 변수안에 저장.

클래스는 자료형 역할을 할 수 있어서 var대신 Test를 넣어도 됨.

참고로 new Test();는 오브젝트라고 부름.

## 사용 - 함수,변수 가져오기
```java
public static void main(String[] args){
  Test test = new Test();

  test.name;
  test.hello();
}
```

저런식으로 .변수, .함수() 로 불러오면 됨.

## class를 쓰는 이유.
1. 자바는 클래스를 써놓고 실행해서 강제로 써야함.
2. 관련있는 함수와 변수끼리 저장해서 정리가 잘 됨.
3. 중요한 변수와 함수를 안전하게 보관가능함. new키워드로 복사해서 쓰기 때문에 비교적 안전함.

## 참고
클래스는 복사안되는 변수나, 복사 안해도 되는 변수를 만들거나 그럴 수 있음.

object는 다른말로 인스턴스.
클래스안에 있는 변수는 field / attribute라고 부기도함.
클래스 안에 있는 함수는 method라고 부르기도 함.

# constructor
```java
class Friend {
  String name = "kim";
  int age = 20;
} 
```

이런 클래스가 있음.
근데 저 이름에 kim이 아니라 park넣고 싶음..
그럼 어떡함? Friend2를 만들면 되긴 하겠지만, 너무 비효율적.

그래서 constructor 문법을 쓸거임.

constructor를 쓰면 new Friend를 할 때마다 다른 값을 줄 수 있음.

## 만들기
```java
class Friend {
  String name;
  int age = 20;
  Friend(String name){
     this.name = name; 
  }
} 
```

클래스안에 클래스이름이랑 똑같은 함수를 넣으면 스페셜한 함수가 됨.

이 스페셜한 함수는 신기하게 this를 쓸 수 있음.
Friend함수 인자(String name)에 name을 넣고,<br>
그 name을 this.name에 주면 이제 "String name"에 이름이 들어감.

# 선언 및 출력
```java
public static void main(String[] args) {
		SpringApplication.run(ShopApplication.class, args);

        Test test1 = new Test("kim");
        System.out.println(test1.name);

        Test test2 = new Test("park");
        System.out.println(test2.name);
	}
```
> 출력
>
> kim<br>
> park

저런식으로 test1, test2각각의 변수에 클래스를 복사하면서, 이름을 다르게 주면 됨.
